LiteLLM 观测站

实时连接 LiteLLM 代理服务，深度监控模型网关性能。提供 Prometheus 指标可视化、并发请求解析、模型用量排行和主动测试能力。
  • 🔍 实时 In‑Flight 并发 / 总请求 / 失败计数
  • 📊 细粒度模型 Token 消耗 (输入/输出/总量) 及占比条形图
  • ⚡ 动态轮询 /metrics 与 /health/backlog，适配任意 LiteLLM 实例
  • 🧪 集成「Test & Measure」抽屉 — 流式评测模型 TTFT、生成吞吐 (tok/s)
  • 🎨 暗色/浅色主题自适应，类终端日志面板
🔌 支持自定义 Endpoint + Master Key，通过 /v1/models 自动获取模型列表。
📍 默认地址: http://localhost:4000
🔎 打开 LiteLLM 监控面板 →

LLM 批量测试台

强大灵活的通用 LLM 基准测试工具，支持 Ollama、LM Studio、llama.cpp、OpenAI、Groq、DeepSeek 等数十种后端。多模型对比、上下文策略、动态队列管理。
  • ✅ 支持 OpenAI 兼容服务 / 本地引擎 (LM Studio, Ollama, llama.cpp, LiteLLM, Jan)
  • 📋 自定义题库 (每行一个问题)，滑动窗口 / 完整上下文 / 无上下文三种模式
  • 🎛️ 多模型选择 & 动态队列: 可中途添加模型、跳过当前模型、移除未测模型
  • ⏱️ 模型间延时控制、实时回答流式渲染、耗时统计及失败处理
  • 📄 结果可下载为 TXT 报告，支持折叠/展开各模型详情，错误高亮
✨ 增强管理: 测试过程中任意加入新模型，跳过出错的模型，保留上下文窗口控制。
🧩 默认对接 LM Studio (http://localhost:1234) 或任意自定义服务。
🧪 启动批量评测 →
🔄 工作流建议
1️⃣ 使用 LiteLLM 观测站 监控代理网关健康度、实时请求量和各模型 Token 消耗。
2️⃣ 搭配 LLM 批量测试台 对候选模型进行标准化问答评测，对比响应质量与耗时。
⚙️ 两者互补: 观测站洞察生产负载，评测台验证模型精度与上下文连贯性。
