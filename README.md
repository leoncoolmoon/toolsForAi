# LLM Workbench

> **LLM 工程工具集**：轻量级、开箱即用的 Web 工具集，专为 LLM 网关监控与模型批量评测设计。

---

## 🎯 项目目的

在开发基于大语言模型（LLM）的应用时，开发者通常需要频繁地对比不同模型的性能、响应质量以及网关的稳定性。**LLM Workbench** 旨在提供一套**无需安装复杂后端**、**纯前端实现**的工具集，帮助开发者：

1.  **实时洞察**：监控模型网关的流量、延迟和 Token 消耗。
2.  **基准评测**：在相同环境下对比多个模型的回答质量与生成速度。
3.  **空间感知**：验证多模态模型对视觉坐标和边界框（BBox）的理解能力。

---

## 🛠️ 核心功能

### 1. 📈 LiteLLM 观测站 (`litellm-performance.html`)
专为 LiteLLM 代理设计的性能监控面板。
-   **实时指标**：可视化并发请求、每秒请求数 (RPS) 和错误率。
-   **用量分析**：详细列出各模型的 Token 消耗（Input/Output/Total）及其占比。
-   **主动测试**：内置测试抽屉，实时测量 TTFT（首字延迟）和生成吞吐量 (tokens/s)。

### 2. 🧪 LLM 批量测试台 (`LLMtester.html`)
通用的多模型对比评测工具。
-   **多后端支持**：兼容 Ollama, LM Studio, llama.cpp, OpenAI, DeepSeek, Anthropic 等。
-   **灵活策略**：支持「无上下文」、「滑动窗口」和「完整上下文」三种对话模式。
-   **动态队列**：测试过程中可随时添加、跳过或移除模型。
-   **结果导出**：支持一键导出详细的评测报告（包含耗时、Token 统计及原始回答）。

### 3. ⬡ BBoxBench 标定台 (`LLM -bbox.html`)
针对多模态模型空间理解能力的专项工具。
-   **视觉标定**：支持 Vision 模式（发送图片）与纯文本模式。
-   **精准度量**：自动计算 IoU（交并比）、中心偏差和面积误差。
-   **多模型对比**：在同一画布上直观对比不同模型的标注差异。

## 🚀 快速开始

### 环境要求
-   **浏览器**：现代浏览器（推荐 Chrome, Edge 或 Firefox）。
-   **模型后端**：本地运行的服务（如 Ollama, LM Studio）或远程 API 密钥。

### 启动方式
由于本工具集是纯 HTML/JS 实现，您可以：
1.  **直接打开**：双击 `index.html`。
2.  **本地服务（推荐）**：使用 Python 快速启动以避免跨域限制。
    ```bash
    python3 -m http.server 8080
    ```
    访问 `http://localhost:8080` 即可进入导航面板。

## 🔐 隐私与安全

-   **本地存储**：所有 API 密钥、端点地址和测试结果均存储在浏览器的 `localStorage` 中。
-   **无数据上传**：本工具不设后端，不会将您的任何敏感信息上传至第三方服务器。
